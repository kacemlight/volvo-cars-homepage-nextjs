# Volvo Cars US Homepage\n\nA modern, content-driven homepage for Volvo Cars built with **NextJS 14+** and **AEM Content Fragments**.\n\n## Overview\n\nThis project fetches structured content from Adobe Experience Manager (AEM) Content Fragments and renders it dynamically on the frontend. All content is authored in AEM and consumed via API — no hardcoded strings.\n\n## Quick Start\n\n### Prerequisites\n- Node.js 18+\n- npm or yarn\n\n### Installation\n\n```bash\ngit clone https://github.com/kacemlight/volvo-cars-homepage-nextjs.git\ncd volvo-cars-homepage-nextjs\nnpm install\n```\n\n### Environment Variables\n\nCreate a `.env.local` file in the root directory with the following variables:\n\n```bash\nAEM_BASE_URL=https://your-aem-instance.adobeaemcloud.com\nAEM_API_KEY=your_aem_api_key_here\nAEM_CONTENT_FRAGMENT_MODEL=volvo-homepage\nAEM_API_TIMEOUT=10000\nNEXT_PUBLIC_ENVIRONMENT=production\n```\n\nRefer to `.env.example` for all available configuration options.\n\n### Development Server\n\n```bash\nnpm run dev\n```\n\nOpen [http://localhost:3000](http://localhost:3000) in your browser.\n\n### Production Build\n\n```bash\nnpm run build\nnpm start\n```\n\n## Project Structure\n\n```\n.\n├── app/                          # NextJS App Router\n│   ├── layout.tsx               # Root layout with metadata\n│   ├── page.tsx                 # Homepage entry point\n│   └── globals.css              # Global Tailwind styles\n├── components/                   # Reusable React components\n│   ├── HeroSection.tsx          # Hero banner section\n│   ├── CarModelCard.tsx         # Individual car model card\n│   ├── FeatureHighlight.tsx     # Feature showcase component\n│   ├── NavigationBar.tsx        # Top navigation\n│   └── FooterSection.tsx        # Footer component\n├── lib/\n│   └── aem.ts                   # AEM Content Fragment API client\n├── public/                       # Static assets\n├── package.json                 # Project dependencies\n├── tsconfig.json               # TypeScript configuration\n├── next.config.js              # NextJS configuration\n├── tailwind.config.ts          # Tailwind CSS configuration\n└── .env.example                # Environment variables template\n```\n\n## Content Fragment Integration\n\nAll page content is fetched from AEM Content Fragments via the API client in `lib/aem.ts`.\n\n### Available Content Fragment Models\n- `volvo-homepage` — Main homepage structure with hero, models, features, footer\n\n### Component Props\n\nEach component accepts content data as props (no hardcoded content). Example:\n\n```typescript\ninterface HeroSectionProps {\n  title: string;\n  subtitle: string;\n  backgroundImage: string;\n  ctaText: string;\n  ctaLink: string;\n}\n```\n\nContent is populated from AEM at build/request time.\n\n## API Client\n\nThe `lib/aem.ts` module provides the following methods:\n\n- `fetchContentFragment(modelName, id)` — Fetch a single fragment\n- `listContentFragments(modelName)` — List all fragments of a model type\n- `getPageContent()` — Fetch full homepage content structure\n\n**Note:** API endpoints and authentication are configured via environment variables.\n\n## Styling\n\nThe project uses **Tailwind CSS** with custom Volvo brand colors:\n- `volvo-blue`: `#003DA5`\n- `volvo-lightblue`: `#0066CC`\n- `volvo-gray`: `#F5F5F5`\n- `volvo-darkgray`: `#333333`\n\n## Responsive Design\n\nAll components are mobile-first and fully responsive for:\n- Mobile (320px+)\n- Tablet (768px+)\n- Desktop (1024px+)\n\n## Deployment\n\nThis project is ready to deploy to Vercel, AWS, or any Node.js hosting platform.\n\n### Vercel (Recommended)\n\n```bash\nnpm install -g vercel\nvercel\n```\n\nSet environment variables in Vercel dashboard.\n\n## Contributing\n\nContent changes are made in AEM.
